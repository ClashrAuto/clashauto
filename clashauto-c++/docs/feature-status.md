# 功能状态清单：已实现 + 待真机验证

> 汇总本轮「对照旧 Electron 版重构」的成果。所有条目均已通过 **GitHub Actions CI 四平台编译**（Windows x64/ARM64 + Linux x64/ARM64），并从**发布产物二进制里 grep 标记字符串确认编入**。
>
> ⚠️ **重要前提**：本机没有 Qt / CMake / MSVC 工具链，且捆绑的 Clash 核心二进制在当前环境无法运行（连 `clash -t` 都崩溃）。因此**没有任何功能在「真实 GUI + 真实核心」下端到端跑过**。验证手段为：静态审查（大括号/捕获/API 用法）＋ 关键逻辑的 PyYAML/urllib 对等移植断言 ＋ CI 编译 ＋ 产物字符串校验。下方「待真机验证」列出仍需在真实机器上确认**运行时行为**的项。

当前最新发布：**v0.1.27**（版本号 = `major.minor` + git 提交数，每次推送 master 自增）。

## 二、待真机验证清单

> 以下均已**编译通过**且逻辑经静态/移植验证，但**运行时行为**需在真实机器（+ 真实 Clash 核心 / 真实订阅 / 真实网络）确认。按优先级排序。

### P0 — 核心链路（必测）
- [ ] **配置注入被核心接受**：带自定义 area/rule 的 `full.yaml` 用真核心 `clash -t -f` 校验通过、`/configs` 热重载成功。（本机核心无法运行，只用 PyYAML 断言了结构正确性）
- [ ] **系统代理 / TUN / 核心生命周期**：启停核心、开关系统代理、TUN 切换、退出还原代理——整条 `CoreController` + `sysproxy.exe` 链路。
- [ ] **GeoIP 落盘路径正确**：核心以 `-f userDir/full.yaml`（无 `-d`）启动，需确认它确实从 `userDir/Country.mmdb` 读取；若读别处需调整落盘路径。

### P1 — 协议 & 单实例（新机制，重点测）
- [ ] **`clash-auto://` 点击链路**：真实 Windows 上注册成功、浏览器/资源管理器点击链接 → 唤起运行中实例 → 导入订阅并拉节点。
- [ ] **单实例转发**：二次启动把参数经命名管道转发给主实例、自身退出、主窗口前置；无主实例时自身成为主实例。
- [ ] **注册表写入**：`reg add` 三条在真实 Windows 成功（尤其 `URL Protocol` 空值那条的兼容性）。

### P2 — REST API / 异步交互
- [ ] **连接查看器**对接真实运行核心的 `/connections`：字段展示、关闭单条/全部、1.5s 实时刷新、搜索过滤。
- [ ] **订阅自动更新定时器**：到点真的触发 `updateSubscription` 并成功拉取（per-sub 周期与全局默认的取模逻辑）。
- [ ] **节点切换通知**：自动 url-test 换节点时通知确实弹出且不在启动误报。
- [ ] **更新弹窗下载**：真实 GitHub 资源下载进度、重定向、落盘、打开文件夹。

### P3 — 本地 UI / 系统交互
- [ ] **跟随系统深浅色**：真实系统切深/浅时应用实时跟随（Qt 6.5+）。
- [ ] **窗口几何记忆**：多显示器/缩放下位置恢复合理、不跑到屏外。
- [ ] **开机自启 / 关闭到托盘**：注册表项生效、开机确实启动；✕ 隐藏到托盘、托盘退出干净。
- [ ] **节点全选 / 导出节点 / 打开配置目录 / 打开面板**：文件写入、外链、目录打开在真实环境正常。

## 三、明确未做 / 平台边界
- **会员 / VIP**：按需求整页移除，不实现。
- **per-sub 更新的持久化时间戳**：当前用「进程内分钟计数取模」（同旧项目），重启后计数归零——与旧项目一致，非缺陷。
- **GeoIP / autoTheme / 开机自启 / 协议注册**：仅 Windows 完整实现；非 Windows 优雅降级或跳过。
- **自更新静默替换重启**：仅做到「下载到目录 + 打开文件夹」，未做下载后自动替换重启。

## 四、如何验证（供后续真机测试者）
```powershell
# 1) 生成配置并用真核心校验（真机上核心可运行时）
.\clashauto-cpp.exe --build-config
<核心路径>\clash-windows-amd64.exe -t -f "%AppData%\ClashAuto\Clash Auto\clash-auto\full.yaml"

# 2) 协议导入（真机 Windows，应用已至少启动过一次完成注册）
start "clash-auto://import?url=<URL编码的订阅地址>&name=测试&type=sub"

# 3) 无 GUI 跑订阅/配置逻辑（CLI 子命令，见 CLAUDE.md / README）
.\clashauto-cpp.exe --list-subscriptions
.\clashauto-cpp.exe --update-subscription 0 C:\path\to\subscription.yaml
```
